# 提交指南.md

## 提交Pull-Request(PR)
**重要**
PR类型
- Bug修复: 添加bug的简要说明(如果有issue则进行关联)
- Feature: 提交的特性必须是kiz标准文档确定的特性, **否则不合并**
- 库: 添加库所有要注册到`kiz std modules`的函数/对象是的简易说明

**重要**
注意编码规范需基本按照kiz-code-style

    ✅ 类名大驼峰, 函数/方法/普通变量/全局变量/别名/命名空间使用小写下划线, Private/Protect属性和局部变量使用名称后下划线命名

    ✅ 4格缩进, 大括号不单独占一行, switch/namespace顶级大括号后不缩进

    ❌ 不常见的缩写

## 提交Bug报告
格式
```markdown
# 标题
===
- 简要说明
- bug复现
```

示例
```markdown
# Decimal除法在遇到负数时无法工作
===
- Decimal除法在遇到负数时无法工作, 直接段错误
- bug复现:
`\`\`kiz repl
>>> 0.1 / - 0.0.1
0.1
`\`\`
```

## 提交功能请求
格式
```markdown
# 标题
===
- 简介
- 所请求的相关语法
```
示例
```markdown
# 关于管道运算符的请求
===
- 管道运算符可以简化嵌套函数调用
- 所请求的相关语法:
`\`\`kiz
expression |> func_name1 |> func_name2
`\`\`
```
